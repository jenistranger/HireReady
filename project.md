## Определение MVP

MVP = публично доступный продукт, которым можно пользоваться без локального запуска.
Не нужно: аккаунты, история, платёжка.
Нужно: не падает под нагрузкой, понятно объясняет что делает, развёрнуто в интернете.

---

## Фаза 1 — Защита (блокирует деплой)

*Цель: не дать сжечь API-ключ и не упасть при первых пользователях*

- [x] **Rate limiting** — slowapi: 5 req/мин на IP для `/api/tailor`, 20 для `/api/pdf`
- [x] **Таймаут у пользователя** — счётчик «Адаптирую... 12с» + предупреждение если >30с
- [x] **Проверка ключа при старте** — startup_check логирует ERROR с инструкцией, endpoint возвращает 500
- [x] **Health check** — `GET /health` возвращает `{"status":"ok","model":"...","api_key_set":true}`
- [x] **CORS** — env var `ALLOWED_ORIGINS`, методы и заголовки сужены, dev-only
- [x] **Лимиты запроса** — Pydantic validators: резюме ≤ 15KB, вакансия ≤ 25KB

**Оценка:** 1 день

## Фаза 2 — UX-минимум (блокирует деплой)

*Цель: новый пользователь понимает что делать за 10 секунд*

- [x] **Онбординг** — placeholder-тексты локализованы для обоих языков
- [x] **Внятные ошибки** — коды 429/504 транслируются в понятные сообщения, все строки в i18n
- [x] **Убрать английские строки** — все JSX-компоненты используют i18n через Context
- [x] **Переключатель языка EN/RU** — кнопка с эмодзи флагами 🇷🇺/🇬🇧, Context-провайдер
- [x] **Шорткат Ctrl+Enter** — запускает генерацию из любого места
- [ ] **Мобильный UX** — проверить на телефоне, фиксить критичные баги
- [ ] **Favicon + title** — `<title>Resume Tailor</title>`, нормальный favicon

**Оценка:** 1 день

## Фаза 3 — Лендинг (блокирует деплой)

*Цель: объяснить продукт до того, как пользователь начнёт вводить данные*

- [ ] **Hero-секция** — заголовок + подзаголовок + CTA «Попробовать бесплатно»
- [ ] **Как это работает** — 3 шага: загрузи резюме → вставь вакансию → получи адаптированное резюме
- [ ] **Пример** — скриншот или живое демо (можно iframe с предзаполненными данными)
- [ ] **Ссылка** — кнопка «Начать» скроллит вниз к форме или переходит на `/app`

Вариант: лендинг и приложение на одной странице (scroll-down), без отдельного роутинга.

**Оценка:** 0.5 дня

---

## Фаза 4 — Деплой

*Цель: работает 24/7 по нормальному домену*

- [ ] **Выбрать платформу** — Railway (проще, дороже) или Fly.io (дешевле, чуть сложнее)
- [ ] **Настроить переменные окружения** — `OPENROUTER_API_KEY`, `DEV_MODE=0`
- [ ] **Настроить домен** — купить или использовать бесплатный поддомен платформы
- [ ] **HTTPS** — автоматически через платформу
- [ ] **Dockerfile проверить** — убедиться что фронтенд билдится и монтируется в `/app/static`
- [ ] **Первый деплой** — smoke test: зайти, вставить тестовое резюме, получить результат, скачать PDF

**Оценка:** 0.5 дня

## После MVP (бэклог)

Не блокирует запуск, добавлять по обратной связи от реальных пользователей:

- **Стриминг ответа** — показывать текст по мере генерации (SSE или streaming API)
- **История без аккаунта** — localStorage: последние 5 адаптаций в браузере
- **Аккаунты** — только если пользователи просят сохранять историю
- **Квота** — только если начнут злоупотреблять (rate limiting сначала)
- **ATS-score** — процент совпадения ключевых слов с вакансией
- **Сопроводительное письмо** — тот же пайплайн, другой промпт
- **Импорт LinkedIn** — парсинг профиля
- **Выбор модели ИИ** — GPT-4o, Claude Haiku как альтернативы
